請依照以下步驟互動式地更新本期臨打活動資訊：

## 步驟一：讀取現有資料

先完整讀取 `index.html`，從中擷取以下共用欄位的目前值：
- 程度（共用資訊卡的程度欄）
- 費用（共用資訊卡的費用欄）
- 報名狀態（Hero 區塊 `<span class='pill' id='status'>` 的文字）

同時記錄目前 `.sessions` 區塊內有幾張場次卡，以及各卡的日期、時間、地點、球場。

## 步驟二：詢問場次數量

**先問這一題，等使用者回覆後才繼續：**

> 這次要推幾場？（目前有 N 場）
> 請輸入場次數（數字），或直接按 Enter 沿用目前場次數：

## 步驟三：逐場詢問日期與時間

依確定的場次數，**每次只問一個問題**，等待使用者回覆後再問下一個：

針對每場（場次一、場次二⋯），依序問：

1. **日期**
   > 場次 N 目前的【日期】是：`目前值`
   > 請輸入新值（格式：YYYY/MM/DD（週），例：2026/07/03（五）），或 Enter 沿用：

2. **時間**
   > 場次 N 目前的【時間】是：`目前值`
   > 請輸入新值（格式：HH:MM - HH:MM），或 Enter 沿用：

3. **地點**（若場次數與現有相同且地點沒變，可整批沿用）
   > 場次 N 目前的【地點】是：`目前值`
   > 請輸入新值，或 Enter 沿用：

4. **球場**
   > 場次 N 目前的【球場】是：`目前值`
   > 請輸入新值，或 Enter 沿用：

若場次數增加，新增的場次以最後一張卡的資料為預設，逐一詢問確認。
若場次數減少，刪除多餘的場次卡。

## 步驟四：詢問共用欄位

場次資訊收集完後，再逐一詢問以下共用欄位：

5. **程度**
6. **費用**
7. **報名狀態**

格式同上（顯示目前值，Enter 沿用）。

## 步驟五：依規則更新

收集完全部回覆後，**只更新有變動的部分**，並遵守 CLAUDE.md 的規則：

- **若場次數有變動**：
  - 在 `.sessions` 內增加或刪除對應的場次卡（照 CLAUDE.md 的場次卡模板格式）
  - 場次卡的標題依序為「場次一」「場次二」⋯

- **若任一場次的日期或時間有變動**，必須同步更新以下四處：
  1. `<title>` 頁面標題中的日期（列出所有場次，如 `07/03・07/06`）
  2. Hero badge（`<span class='badge'>` 內，列出所有場次日期）
  3. `.sessions` 內對應場次卡的日期欄
  4. 底部 `<script>` 的 `sessions` 陣列（格式：`YYYY-MM-DDTHH:MM:00+08:00`，依開始時間排序）

- 其餘欄位只改對應的 `<div class='value'>` 或 `<span>` 內容。

## 步驟六：自查與驗證

執行 `git diff index.html`，確認：
- 只有活動資料欄位被修改，沒有動到樣式或固定文案

確認頁面仍包含以下三個關鍵字串：
- `草莓羽球團幫`
- `我要報名`
- `常見問題`

## 步驟七：Commit 與 Push

若驗證全部通過，依序執行：
```
git add index.html
git commit -m "Update meetup event information"
git push
```

若驗證未過，**停止、不要 commit**，直接告知哪裡有問題。

## 步驟八：回報結果

列出所有有變動的欄位（新舊值對照表），以及 commit hash 與 push 結果。
未變動的欄位不用列出。
